## 1. KG이니시스

- **공식 사이트:** https://www.inicis.com
- **개발자 문서:** https://manual.inicis.com
- **정기결제 문서:** https://manual.inicis.com/pay/bill_pc.html
- **정기결제 방식:** 빌링키 (결제창 / REST API)
- **지원 결제수단:** 신용카드, 체크카드 (정기결제 시 카드만 지원)

### 장점
- 국내 최대 PG사 중 하나 — 안정성, 레퍼런스 풍부
- 다양한 부가서비스 (통합인증, 간편결제 WPAY)
- 많은 은행/카드사 연동 폭

### 단점
- REST API 방식 빌링키 발급은 별도 계약 및 입점 심사 필요 (까다로움)
- 초기 가입비 약 22만원
- 문서 구조가 다소 복잡하고 오래된 편

### SaaS 구독 적합도
카드 기반 구독 결제에는 충분하나, API 직접 방식은 심사 허들이 존재.

## 2. NHN KCP

- **공식 사이트:** https://kcp.co.kr
- **개발자 센터:** https://developer.kcp.co.kr
- **정기결제 문서:** https://developer.kcp.co.kr/page/document/autopay
- **정기결제 방식:** 배치키 (결제창 + REST API 모두 지원)
- **지원 결제수단:** 신용카드, 체크카드만 지원

### 장점
- 국내 상위 4대 PG사, 안정적인 서비스 운영
- 개발자 문서 충실하고 정기구독 특화 기능 있음
- KCP PAY 간편결제 내장 연동 가능
- 결제창 / API 방식 모두 지원

### 단점
- 자동결제는 카드만 지원 (계좌이체 불가)
- 별도 계약 필요
- 수수료: 첫 1년 2.9%, 이후 3.2% (비교적 높은 편)

### SaaS 구독 적합도
카드 구독 결제에 최적화. 정기구독 레퍼런스 다수 보유.

## 3. NICE페이먼츠

- **공식 사이트:** https://www.nicepay.co.kr
- **개발자 센터:** https://developers.nicepay.co.kr
- **빌링 API 문서:** https://developers.nicepay.co.kr/manual-card-billing.php
- **GitHub 매뉴얼:** https://github.com/nicepayments/nicepay-manual
- **정기결제 방식:** 빌키(BID) — REST API 직접 방식
- **지원 결제수단:** 카드, 계좌 CMS(더빌), 가상계좌

### 장점
- 카드 빌링 외 계좌 자동이체(CMS) 전문 서비스 **더빌** 별도 운영
- GitHub에 공개 매뉴얼 — JSP/PHP/Node.js/Python 샘플 코드 제공
- 해외 카드 결제 지원 (3.8%)
- 다양한 결제 수단 통합 가능

### 단점
- 베이직 플랜 등록비 33만원 + 수수료 3.5% (높음)
- 스페셜 플랜은 등록비 55만원 (초기 비용 부담)
- API 직접 방식이라 개발 공수 필요

### SaaS 구독 적합도
계좌이체 기반 구독도 함께 지원하고 싶다면 강점. 해외 결제 확장도 고려할 경우 유리.

## 4. 카카오페이

- **공식 사이트:** https://www.kakaopay.com
- **개발자 센터:** https://developers.kakaopay.com
- **정기결제 문서:** https://developers.kakaopay.com/docs/payment/online/subscription
- **정기결제 방식:** SID (Subscription ID) 방식
- **지원 결제수단:** 카카오페이머니, 카카오페이 등록 카드

### 장점
- 카카오톡 사용자 기반으로 높은 전환율 기대
- 0원 SID 발급 → 이후 결제 방식 지원으로 무료 체험 구독 구현 용이
- UX/UI 우수, 개발자 포럼 운영

### 단점
- **카카오페이 사용자만 결제 가능** (비가입자 불가)
- 할부 불가
- 별도 가맹점 계약 + 검수 과정 필요
- 수수료 비공개

### SaaS 구독 적합도
카카오 생태계 내 사용자 전환에는 강하나, 결제 수단이 카카오페이로 제한되어 단독 사용 시 이탈률 위험.

## 5. 네이버페이

- **공식 사이트:** https://pay.naver.com
- **개발자 센터:** https://developer.pay.naver.com
- **정기결제 방식:** recurrentId 방식
- **지원 결제수단:** 네이버페이 등록 카드, 네이버 포인트

### 장점
- 네이버 검색/쇼핑 생태계 연동으로 노출 효과
- PG사 중간 수수료 없음 (네이버페이 수수료만 부과)
- 일반결제 + 정기결제 통합 지원

### 단점
- **네이버페이 사용자만 결제 가능**
- 검수 기간 평균 15영업일 (긴 편)
- 주문형/결제형 구분 필요 (설계 복잡도 증가)
- 수수료 1.87%~3.74% (연매출 구간별)

### SaaS 구독 적합도
네이버 스마트스토어 기반 판매와 병행하는 경우 시너지. 독립 SaaS 구독에는 카카오페이와 동일하게 사용자 제한 문제 존재.

## 6. 페이코

- **공식 사이트:** https://www.payco.com
- **개발자 센터:** https://devcenter.payco.com
- **정기결제 방식:** 자동결제 등록 (결제창 방식)
- **지원 결제수단:** PAYCO 포인트, 등록 카드, 계좌이체

### 장점
- NHN 계열사로 NHN KCP와 결합 운영 가능
- 고도몰 등 NHN커머스 플랫폼과 연동 용이
- PAYCO 포인트 기반 혜택 제공으로 사용자 리텐션 가능

### 단점
- 카카오페이·네이버페이 대비 사용자 수 적음
- 전용 정기결제 API 문서 외부 공개 부족
- 포트원 등 미들웨어 의존도 높음

### SaaS 구독 적합도
NHN 생태계 내 서비스에는 시너지. 독립 SaaS 구독으로는 레버리지가 제한적.

## 7. 스마트로

- **공식 사이트:** https://www.smartro.co.kr
- **공식 매뉴얼:** https://manual.smartropay.co.kr
- **빌링 문서:** https://manual.smartropay.co.kr/unionbill.do
- **정기결제 방식:** 빌링키 (신용카드 통합 빌링)
- **지원 결제수단:** 신용카드, 계좌이체 (2024년 4분기 추가)

### 장점
- 신규/기존 빌링 단일 API로 처리
- 금융결제원과 MOU — 계좌 자동이체 기반 정기결제 추가 (저수수료)
- 다양한 언어 샘플 코드 제공 (JSP/PHP/ASP/Node.js/Python)
- 오프라인 + 온라인 통합 결제 가능

### 단점
- 빌링키 발급 시 **휴대폰 본인인증 필수** (UX 마찰)
- customer_id 필수 입력
- 결제창 한국어만 지원 (글로벌 확장 시 제한)
- 수수료 비공개

### SaaS 구독 적합도
계좌이체 정기결제를 저수수료로 도입하고 싶다면 향후 유력 후보. 현재는 카드 중심.

## 8. 헥토파이낸셜 (구 세틀뱅크)

- **공식 사이트:** https://www.hectofinancial.co.kr
- **개발자 사이트:** https://develop.sbsvc.online
- **신규 개발자 사이트 (2026.05.15 이후):** https://developers.hectofinancial.co.kr
- **정기결제 방식:** 빌링키 (REST API 방식만 지원)
- **지원 결제수단:** 카드, 계좌이체, 가상계좌, 휴대폰, 계좌 자동이체

### 장점
- **가상계좌 국내 1위** 서비스
- 계좌 기반 정기 자동이체(펌뱅킹)에 강점
- 다양한 결제 수단 통합 지원
- ARS 출금이체 동의 / 계좌점유인증 등 안전거래 특화
- SMS/알림톡/이메일 알림 대행 서비스 내장

### 단점
- 정기결제 시 **REST API 방식만 지원** (결제창 방식 없음 — 직접 개발 필요)
- 빌링키 단독 발급은 별도 계약 필요
- 현재 개발자 문서 사이트 이전 중 (2026.05.15)

### SaaS 구독 적합도
계좌이체 기반 정기구독(B2B 청구 등)에 강점. 카드 구독만 필요하면 다른 PG사가 더 간단.

## 9. 웰컴페이먼츠

- **공식 사이트:** https://www.welcomepayments.co.kr
- **포트원 연동 문서:** https://developers.portone.io/opi/ko/integration/pg/v1/welcome/readme?v=v1
- **정기결제 방식:** 빌링키 (결제창 방식)
- **지원 결제수단:** 카드, 휴대폰

### 장점
- 웰컴금융그룹 계열 (웰컴저축은행 연계)
- 비대면 카드결제 특화
- 카드번호 직접 결제(비인증) 지원
- 에스크로 결제 지원

### 단점
- 타 대형 PG사 대비 인지도 낮음
- 별도 정기결제 개발자 문서 공개 부족
- 포트원 V1 기반 (신규 연동 시 V2 전환 권장)
- 전문 개발자 지원 창구 제한적

### SaaS 구독 적합도
비대면 카드 정기결제는 가능하나, 레퍼런스와 문서가 부족해 도입 리스크 존재.

## 10. 갤럭시아머니트리 (빌게이트)

- **공식 사이트:** https://www.galaxiamoneytree.co.kr
- **빌게이트:** https://www.billgate.net
- **정기결제 방식:** 빌링키 / 전자결제 (API 문서 비공개)
- **지원 결제수단:** 카드, 휴대폰, 계좌이체

### 장점
- 갤럭시아그룹 계열 — 미디어·엔터테인먼트 결제 특화
- 사이버머니(MONEYTREE) + PG + B2B 쿠폰 통합 솔루션
- 온·오프라인 통합 전자결제 가능

### 단점
- **공개 API 문서 없음** — 가맹 계약 후에만 제공
- 정기결제 특화 기능 확인 어려움
- 타 대형 PG사 대비 시장인지도 낮음
- 검토 단계에서 정보 수집 자체가 제한적

### SaaS 구독 적합도
엔터테인먼트/미디어 분야 특화라 영상제작 SaaS와 결은 맞으나, 문서 비공개로 연동 진입장벽 높음.

## SaaS 구독 서비스 도입 추천 순위

1. **NICE페이먼츠** — 카드 + 계좌이체 동시 지원, GitHub 공개 문서, 샘플 코드 풍부
2. **KG이니시스 / NHN KCP** — 국내 최대 레퍼런스, 안정성 최상
3. **스마트로** — 계좌 자동이체 추가로 결제 수단 다양화 진행 중
4. **헥토파이낸셜** — 계좌이체 기반 B2B 구독 청구에 특화
5. **카카오페이 / 네이버페이** — 보조 결제 수단으로 병행 도입 (단독 사용은 비추천)
